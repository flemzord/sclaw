sclaw is a **plugin-first, self-hosted personal AI assistant** built in Go. Inspired by [Caddy](https://caddyserver.com/)'s modular architecture, sclaw treats everything as a module — channels, providers, memory backends, and tools are all pluggable components that can be mixed, matched, and extended.

## Why sclaw?

Most AI assistant frameworks are either too rigid (locked to a single provider) or too complex (requiring deep infrastructure knowledge). sclaw sits in the sweet spot:

- **Single binary, zero dependencies** — Download and run. No Docker, no database server, no runtime.
- **Plugin-first architecture** — Every component is a module. Swap providers, add channels, or extend functionality without forking.
- **Config-driven** — A single YAML file controls everything. No code changes needed for most customizations.
- **Defense-in-depth security** — Rate limiting, credential isolation, tool sandboxing, and audit logging built in from day one.
- **Multi-agent routing** — Run multiple agents with different personalities, tools, and providers, routed by channel, user, or group.

## Core Architecture

sclaw's core handles orchestration while modules provide the actual capabilities:

| Component | Responsibility |
|-----------|---------------|
| **Module Registry** | Discovers and wires modules at startup |
| **Router** | 15-step message pipeline with session management |
| **Agent Loop** | ReAct reasoning with tool execution |
| **Context Engine** | Token budget management and history compaction |
| **Provider Chain** | Model failover with health tracking |
| **Memory System** | Two-tier recall (conversation + long-term facts) |
