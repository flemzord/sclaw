The `provider.openai_responses` module connects sclaw to OpenAI's [Responses API](https://platform.openai.com/docs/api-reference/responses) using native WebSocket streaming. Compared to the REST+SSE approach of `provider.openai_compatible`, WebSocket provides a persistent connection with lower latency — no reconnection per request.

<Note>
This provider is **OpenAI-specific**. For third-party APIs (Ollama, Azure, vLLM, etc.) that implement the Chat Completions interface, use [`provider.openai_compatible`](/modules/providers/openai-compatible) instead.
</Note>

## Configuration

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `api_key` | string | — | API key. One of `api_key` or `api_key_env` is **required**. |
| `api_key_env` | string | — | Environment variable name containing the API key. |
| `model` | string | — | Model identifier (e.g., `"gpt-4o"`). **Required.** |
| `context_window` | int | `128000` | Maximum context window size in tokens. |
| `max_output_tokens` | int | `0` | Maximum tokens to generate (0 = provider default). |
| `ws_endpoint` | string | `"wss://api.openai.com/v1/responses"` | WebSocket endpoint URL. |
| `headers` | map | — | Extra HTTP headers sent during WebSocket handshake. |
| `dial_timeout` | duration | `10s` | Timeout for establishing a WebSocket connection. |
| `conn_max_age` | duration | `55m` | Maximum connection lifetime before recycling (OpenAI limit: 60min). |

## Authentication

The provider supports the same two authentication methods as `provider.openai_compatible`:

### Direct API Key

Embed the key directly (with environment variable or 1Password expansion):

```yaml
api_key: "${OPENAI_API_KEY}"
```

### Environment Variable Reference

Reference an environment variable by name:

```yaml
api_key_env: "OPENAI_API_KEY"
```

## WebSocket Connection Lifecycle

The provider manages a single persistent WebSocket connection:

1. **Lazy dial** — Connection is established on the first request, not at startup.
2. **Reuse** — The same connection is reused across requests (one at a time).
3. **Max age recycling** — Connections older than `conn_max_age` are replaced. OpenAI enforces a 60-minute limit; the default of 55 minutes provides a safety margin.
4. **Auto-reconnect** — If a connection is lost (server error, network issue), a fresh one is dialed on the next request.

<Warning>
The Responses API allows only **one response at a time** per WebSocket connection. Since the agent loop is sequential, this is not a limitation in practice. For parallel agent execution, each agent gets its own provider instance.
</Warning>

## Streaming

All responses are streamed natively over the WebSocket. The provider:

1. Sends a `response.create` event with the model, instructions, and conversation input
2. Receives streaming deltas for text and tool call arguments
3. Accumulates tool call fragments by output index
4. Reports token usage and finish reason when the response completes

## Comparison with openai_compatible

| Feature | `openai_compatible` | `openai_responses` |
|---------|--------------------|--------------------|
| Protocol | REST + SSE | WebSocket |
| Connection | New HTTP request per call | Persistent, reused |
| API | Chat Completions (`/v1/chat/completions`) | Responses (`/v1/responses`) |
| Compatibility | Any OpenAI-compatible API | OpenAI only |
| Latency | Higher (TCP+TLS per request) | Lower (persistent connection) |

## Context Window

Set `context_window` to match your model's maximum context size. The default of 128,000 tokens matches GPT-4o.

```yaml
context_window: 200000  # For models with larger context windows
```

## Health Check

The provider probes the `/v1/models` REST endpoint for health checks, using the same API key. This is used by the provider chain to track availability and trigger failover.
